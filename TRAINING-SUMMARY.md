# Training Plan Summary

Quick overview of the complete training program for getting a non-technical person up and running with promptfoo.

## 📋 At a Glance

| Aspect | Details |
|--------|---------|
| **Target Audience** | Non-technical, Windows, very comfortable with new tools |
| **Total Time** | 15-20 hours |
| **Format** | 6 in-person sessions + self-study |
| **Approach** | VSCode + YAML + integrated terminal |
| **Prerequisites** | Admin rights (or PEDM access) |

## 🎯 Learning Objectives

After completing this training, the trainee will be able to:

1. ✅ Install and configure VSCode, Node.js, and Git
2. ✅ Create and edit YAML configuration files
3. ✅ Write promptfoo evaluation configurations
4. ✅ Run evaluations and interpret results
5. ✅ Use Git to manage their work
6. ✅ Troubleshoot common issues independently

## 📅 Training Structure

### Phase 1: Setup & Basics (4-6 hours)

**Session 1: Installation & First Steps (2-3 hours)**
- Install VSCode, Node.js, Git
- VSCode interface tour
- First terminal commands

**Session 2: YAML & Git Fundamentals (2-3 hours)**
- YAML syntax and patterns
- Git basics in VSCode
- Create and commit first file

### Phase 2: Promptfoo Fundamentals (6-8 hours)

**Session 3: Understanding Config Structure (3-4 hours)**
- Walk through existing config
- Run first evaluation
- Simple modifications

**Session 4: Creating New Tests (3-4 hours)**
- Test case patterns
- Assertion types
- Hands-on practice

### Phase 3: Working Independently (4-6 hours)

**Session 5: Advanced Patterns (2-3 hours)**
- defaultTest and templates
- Organization strategies
- Transforms

**Session 6: Troubleshooting & Best Practices (2-3 hours)**
- Common errors
- Debugging strategy
- Create personal cheat sheet

### Phase 4: Self-Study (4-6 hours)

**Homework Assignments:**
1. Complete eval config (2 hours)
2. Modify existing tests (1 hour)
3. Git workflow practice (1 hour)
4. Results analysis (1-2 hours)

**Optional Follow-up:** 1-2 hours

## 💡 Key Decisions Made

### Why VSCode + YAML?
- ✅ VSCode has excellent YAML support
- ✅ Integrated terminal (no separate app)
- ✅ Git integration built-in
- ✅ One tool for everything
- ✅ Trainee is comfortable learning new tools
- ✅ Transferrable skills

### Why not CSV/Excel?
- YAML is more flexible for complex configs
- Trainee can handle the initial complexity
- Better for learning promptfoo concepts
- CSV can be introduced later for bulk tests

### Why this approach?
- Leverages trainee's comfort with new tools
- Provides transferrable skills
- VSCode handles complexity (validation, formatting)
- Node.js installation is straightforward on Windows
- Git integration prepares for collaboration
